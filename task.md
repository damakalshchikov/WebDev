# Лабораторная работа №3
Необходимо создать базу данных *(PostgreSQL)* для товаров, отзывов и пользователей. Для пользователей также нужно реализовать регистрацию/авторизацию и валидацию введённых им данных для авторизации/регистрации.
## Цель

Перевести сайт «ДИАМАНД» с захардкоженных данных (`src/data/products.js`) на реальную БД PostgreSQL. Реализовать рабочий бэкенд на Node.js/Express, к которому фронт обращается через `fetch`.

---

## 1. База данных (PostgreSQL)

### Обязательные таблицы

| Таблица | Что хранит |
|---|---|
| `users` | Пользователи и администраторы |
| `products` | Украшения (карточки каталога) |
| `product_properties` | Доп. свойства украшений (материал, проба и т.п.) |
| `product_images` | Доп. изображения к украшению |
| `reviews` | Отзывы покупателей |
| `messages` | Письма (контакты / запись на примерку) |
| `gift_cards` | Подарочные карты |
| `gift_card_reservations` | Резервирования подарочных карт |

> Структура `products` берётся из методички ЛР3 (`id`, `name`, `short_description`, `description`, `price`, `image`, `available`, `category` и т.д.).

## 2. Структура Node.js-бэкенда

Рекомендуемая структура проекта (отдельная папка рядом с фронтом, например `/server`):

```
server/
├── config/
│   └── db.js            # подключение к PostgreSQL
├── controllers/
│   ├── authController.js
│   ├── productsController.js
│   ├── reviewsController.js
│   ├── messagesController.js
│   └── giftCardsController.js
├── middleware/
│   └── auth.js          # проверка cookie-сессии
├── routes/
│   ├── auth.js
│   ├── products.js
│   ├── reviews.js
│   ├── messages.js
│   └── giftCards.js
└── server.js
```

Зависимости: `express`, `pg`, `bcrypt`, `cookie-session` (или `express-session`), `cors`.

В `vite.config.js` настроить **proxy** на бэкенд:

```js
server: {
  proxy: {
    '/api': 'http://localhost:3000'
  }
}
```

Тогда фронт делает `fetch('/api/products')` без CORS-проблем.

## 3. Каталог украшений (перевод на БД)

- Убрать импорт `products` из `src/data/products.js` — заменить на `fetch('/api/products')`.
- `Catalog.jsx` загружает категории и товары с бэкенда.
- `ProductDetail.jsx` загружает конкретный товар по `id` с бэкенда.

- Структура каталога: **3 категории** (золото / серебро / платина), в каждой минимум **3 товара** — данные в БД.

- Форма добавления включает: название, краткое описание, полное описание, материал, цену, изображение, категорию.
- На странице `ProductDetail` у администратора отображаются кнопки **«Редактировать»** и **«Удалить»**.

## 4. Аутентификация

### 4.1 Регистрация (`/register`)

- Форма уже есть в `Register.jsx` — подключить к `POST /api/auth/register`.
- Пароль хешируется через `bcrypt` перед записью в БД.
- Поля: имя, телефон (`+7XXXXXXXXXX`), email, пароль × 2, два чекбокса (ПДн, политика).

### 4.2 Вход

- Форма уже есть в шапке (`Header.jsx`) — подключить к `POST /api/auth/login`.
- При успешном входе сервер выставляет **cookie** с сессией.
- Состояние текущего пользователя хранить через `React Context`.

### 4.3 Поведение шапки после входа

| Состояние | Что показывать |

## 5. Отзывы (`/reviews`)

Форма отзыва уже есть в `Reviews.jsx` — подключить к бэкенду.

### Жизненный цикл отзыва

```
Пользователь отправил → POST /api/reviews
        ↓
  status = 'pending'  →  виден только администратору
        ↓
  Администратор одобряет → PATCH /api/reviews/:id/approve
        ↓
  status = 'approved'  →  виден всем посетителям
```

- Захардкоженные отзывы из `Reviews.jsx` **удалить** — отображать только записи из БД со статусом `approved`.
- Пользователь **может редактировать** свой одобренный отзыв (`PUT /api/reviews/:id`).
- Пользователь **не может удалять** свои отзывы.
- Администратор **может удалять** любые отзывы и одобрять pending-отзывы.
- Администратор **не может** оставлять собственные отзывы.

## 6. Сообщения

Плавающая кнопка «✉» в правом нижнем углу — отображается над футером только у авторизованных пользователей и администратора.  
При нажатии открывается **всплывающее окно** (не во весь экран), закрывается крестиком.

### Откуда поступают письма

1. **`Contacts.jsx`** — форма «Напишите нам». Подключить к `POST /api/messages`.
2. **`ProductDetail.jsx`** — кнопка «Записаться на примерку» (видна только авторизованному пользователю, не администратору). При нажатии — модальное окно → `POST /api/messages`.

### Поведение по ролям

| | Пользователь | Администратор |
|---|---|---|
| Написать новое письмо | ✅ (через контакты или товар) | ❌ |
| Видеть свои отправленные письма и ответы | ✅ | — |
| Видеть все входящие письма | — | ✅ |
| Ответить на письмо | ❌ | ✅ |

## 7. Подарочные карты

- Хранятся в таблице `gift_cards`.
- Администратор может **добавлять** и **удалять** карты.

## 10. API — список маршрутов

| Маршрут | Метод | Описание | Кто |
|---|---|---|---|
| `/api/auth/register` | POST | Регистрация | — |
| `/api/auth/login` | POST | Вход | — |
| `/api/auth/logout` | POST | Выход | — |
| `/api/auth/me` | GET | Текущий пользователь из cookie | — |
| `/api/products` | GET | Список товаров (`?search=`, `?category=`) | — |
| `/api/products/:id` | GET | Один товар | — |
| `/api/products` | POST | Добавить товар | admin |
| `/api/products/:id` | PUT | Редактировать товар | admin |
| `/api/products/:id` | DELETE | Удалить товар | admin |
| `/api/reviews` | GET | Одобренные отзывы | — |
| `/api/reviews` | POST | Оставить отзыв | user |
| `/api/reviews/:id` | PUT | Редактировать свой отзыв | user |
| `/api/reviews/pending` | GET | Отзывы на модерации | admin |
| `/api/reviews/:id/approve` | PATCH | Одобрить отзыв | admin |
| `/api/reviews/:id` | DELETE | Удалить отзыв | admin |
| `/api/messages` | GET | Свои письма / все письма | user/admin |
| `/api/messages` | POST | Отправить письмо | user |
| `/api/messages/:id/reply` | POST | Ответить на письмо | admin |
| `/api/gift-cards` | GET | Список карт | — |
| `/api/gift-cards` | POST | Добавить карту | admin |
| `/api/gift-cards/:id` | DELETE | Удалить карту | admin |
| `/api/gift-cards/:id/reserve` | POST | Зарезервировать карту | user |
